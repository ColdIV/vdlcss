1.10 Dialogs

Dialogs require javascript to be rendered. A simple example would look like this:

<dialog>
    <p>Example dialog</p>
    <form method="dialog">
        <button class="m-0">Close</button>
    </form>
</dialog>

<script>
    function openExampleDialog() {
        document.querySelector('#example-dialog').open = true;
    }
</script>

<button class="mt-1" onclick="openExampleDialog()">Show Dialog</button>

2.0 Examples with Class

2.1 Colors

Colors can be changed and accessed via CSS-Variables.

To use colors as text color, just add the color name as class. To use it as background color, prefix it with bg-.

Green [class="bg-green white p-1"]
Blue [class="bg-blue white p-1"]
Yellow [class="bg-yellow white p-1"]
White [class="bg-white black p-1"]
Gray [class="bg-gray white p-1"]
Black [class="bg-black white p-1"]

2.6 Container

Containers [class="container"] center and add a max-width to the div as well as some base padding on the left and right side. The max-width is based on the viewport. You can find the values here.

2.7 Grid

The grid system is a 12-column System. We can have a grid that wraps multiple cells. Cells can have a size from 1 to 12.

[class="cell-12 md:cell-4 border p-1"]
[class="cell-12 md:cell-8 border p-1"]
[class="cell-12 border p-1"]

You can also use column-start-x, column-end-x, row-start-x and row-end-x. x ranges from 1 to 13.

column-start-1 column-end-5 border p-1
column-start-7 column-end-13 border p-1
column-start-3 column-end-7 border p-1

Note: Grid classes support viewport-prefixes.

2.8 Rows and Columns

Rows and columns are used to position elements using flex without too much typing.

Classes like left, right, top, bottom and center work for columns and rows.

Wrapper has [class="row left my-1 border"]
Wrapper has [class="row right my-1 border"]
Wrapper has [class="row center-horizontal my-1 border"]
Wrapper has [class="row center-vertical my-1 border" style="height: 200px;"]
Wrapper has [class="row center my-1 border" style="height: 200px;"]
Wrapper has [class="row top my-1 border" style="height: 200px;"]
Wrapper has [class="row bottom my-1 border" style="height: 200px;"]

The difference between row and column is, within a row, content will be placed underneath each other. Whereas within a column, content will be placed side-to-side.

Wrapper has [class="row left my-1 border"]
Elements have [class="bg-black white p-1 mx-05"]
Wrapper has [class="column left my-1 border"]
Elements have [class="bg-black white p-1 my-05"]

Content can easily be positioned with between, evenly and around. They work on both, row and column.

Wrapper has [class="row between my-1 border"]
Elements have [class="bg-black white p-1 mx-05"]
Wrapper has [class="row evenly my-1 border"]
Elements have [class="bg-black white p-1 mx-05"]
Wrapper has [class="row around my-1 border"]
Elements have [class="bg-black white p-1 mx-05"]

Note: Row and column support viewport-prefixes.

2.11 Scroll and Scrollbar

You can allow scrolling with either scroll-x, scroll-y or with snap, scroll-x-snap and scroll-y-snap. If you do not want any overflow, then overflow-hidden is what you are looking for.

To have a styled scrollbar, add the class custom-scrollbar to the wrapping element.

2.12 Padding and Margin

There are many classes for padding and margin. They all follow the same pattern, a prefix followed by a value. The values always range from 0 to 5 with two extra values being 025 and 05.

The valid prefixes are: p, m, pt, pr, pb, pl, mt, mr, mb, ml, px, py, mx, my.

p always means padding, m always margin. t, r, b and l are short for top, right, bottom and left. And x and y are short for x- and y-axis.

Additionaly there are the classes ml-, mx- and mr-auto, which sets the margin for the given direction to auto.

Note: Padding and margin support viewport-prefixes.

2.13 Width and height

You can set width and height to either auto or 100% with w-auto and w-100.

Note: Width and height support viewport-prefixes.

2.14 Viewport-Prefixes

Viewport-prefixes can be used for some specific classes. If you add a viewport-prefix to them, they will only work from that viewport upwards.

A simple example is this grid, where the cells have the following classes

cell-12 md:cell-6 border

On small devices, the cells will fill the whole width. On bigger devices, they only fill half of it. You can find the breakpoints and the prefix names here.
